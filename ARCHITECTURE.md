## Технологический стек

- **Framework**: Next.js 16 (App Router)
- **Language**: TypeScript
- **Styling**: Tailwind CSS 4
- **React**: 19.2.3
- **Package Manager**: Yarn

## Принципы архитектуры

### 1. Модульность
- Каждая фича изолирована в своей директории
- Компоненты разделены на UI (переиспользуемые) и feature-specific

### 2. Типизация
- Все компоненты и функции типизированы через TypeScript
- Общие типы хранятся в `src/types/`

### 3. Переиспользуемость
- UI компоненты (`src/components/ui/`) могут использоваться в любых фичах
- Утилиты (`src/lib/`) - общие функции для всего приложения

### 4. Константы
- Все данные о свадьбе централизованы в `src/constants/wedding.ts`
- Легко изменяемая конфигурация

## Основные фичи

### Hero Section
Главный экран с именами, датой и призывами к действию.

### Countdown
Динамический таймер обратного отсчета до дня свадьбы.

### Details
Информация о времени и месте проведения.

### RSVP
Форма для подтверждения участия гостей.

### Gallery
Галерея фотографий пары.

### Footer
Нижняя часть сайта с информацией.

## Как добавить новую фичу

1. Создайте директорию в `src/features/новая-фича/`
2. Создайте основной компонент `НоваяФича.tsx`
3. При необходимости добавьте типы в `src/types/`
4. Добавьте константы в `src/constants/`
5. Импортируйте и используйте в `app/page.tsx`

## Стилизация

- Используем Tailwind CSS для всех стилей
- Responsive-first подход (mobile → desktop)
- Цветовая схема: Rose (основной) + Gray (текст/фон)
- Утилита `cn()` для условных классов
